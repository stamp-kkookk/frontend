# 꾸욱(KKOOKK) 랜딩 페이지 가이드라인

## 1. 개요 (Overview)

이 문서는 '꾸욱(KKOOKK)' 서비스의 랜딩 페이지 제작을 위한 가이드라인을 제공합니다. '꾸욱'은 POS 연동이 필요 없는 웹 기반 디지털 스탬프/쿠폰 플랫폼으로, 소상공인(사장님)들이 손쉽게 고객 재방문율을 높일 수 있도록 돕는 SaaS입니다. 랜딩 페이지의 궁극적인 목표는 **사장님들의 회원가입 및 서비스 도입**을 유도하는 것입니다.

---

## 2. 핵심 메시지 및 타겟 (Core Message & Target Audience)

### 2.1 타겟 고객: 소상공인 사장님 (Owner Persona)

랜딩 페이지의 모든 콘텐츠와 디자인은 '바쁜 사장님'의 입장에서 '꾸욱' 서비스가 제공하는 가치를 명확하고 간결하게 전달해야 합니다.

### 2.2 핵심 가치 제안 (Unique Value Proposition)

- **"POS 연동 없는"**: 복잡한 설치나 연동 없이 바로 사용 가능한 간편함 강조.
- **"앱 설치 없는"**: 고객이 앱을 설치할 필요 없이 QR 스캔만으로 즉시 이용 가능함을 강조.
- **"매장 승인형 디지털 스탬프"**: 사장님의 통제 하에 운영되는 안전하고 직관적인 시스템임을 강조.
- **"기존 종이 스탬프 마이그레이션"**: 서비스 전환의 진입 장벽을 낮추는 독점적인 기능 강조.
- **"재방문율 증가"**: 실제 사장님들의 매출 증대로 이어질 수 있는 핵심 이점 강조.

### 2.3 커뮤니케이션 톤앤매너

사장님들의 공감을 얻을 수 있도록 친근하고, 신뢰성 있으며, 문제 해결에 집중하는 톤을 유지합니다. 기술적인 용어보다는 실질적인 비즈니스 이점과 간편함을 중심으로 소통합니다.

## 3. 페이지 구조 (Page Structure)

랜딩 페이지는 다음 순서로 구성하여 방문자의 궁금증을 해소하고 자연스럽게 서비스의 가치를 경험하도록 유도합니다.

### 3.1 헤더 (Header)

- **구성:** 왼쪽 상단에 '꾸욱' 로고, 오른쪽 상단에 "문의하기" CTA 버튼.
- **목표:** 가장 기본적인 탐색과 문의 채널을 간결하게 제공합니다.

### 3.2 히어로 섹션 (Hero Section)

- **핵심 메시지:** 강렬한 헤드라인으로 방문자의 시선을 사로잡습니다.
- **CTA:** "스탬프 생성하러 가기" 버튼을 통해 즉각적인 행동을 유도합니다.
- **시각 자료:** 실제 종이 스탬프가 '꾸욱'의 디지털 스탬프로 변환되는 모습을 캐러셀(Carousel) 형태로 보여주어, 마이그레이션 기능의 가치를 직관적으로 전달합니다.

### 3.3 Problem 섹션

- **공감대 형성:** "이런 고민 너무 익숙하시죠?"와 같은 질문을 통해 사장님들이 겪는 일반적인 문제(종이 쿠폰 분실, 관리의 어려움, 재방문 유도 실패 등)를 제시하여 공감대를 형성합니다.

### 3.4 Solving 섹션

- **해결책 제시:** "매출이 오르는 스탬프, 이렇게 만듭니다"라는 헤드라인 아래, '꾸욱'의 스탬프 커스터마이징 과정을 UX Flow Preview 형태로 보여줍니다.
- **가치 전달:** 방문자가 직접 스탬프를 만드는 과정을 시각적으로 경험하게 하여, 서비스의 간편함과 강력한 기능을 체감하게 합니다.

### 3.5 보안 섹션 (Security Section)

- **신뢰 구축:** "걱정 없이 시작하세요"라는 메시지와 함께, '꾸욱'의 신뢰성을 뒷받침하는 기능들을 시각적으로 제시합니다.
  - **리딤 감사/로그 페이지:** 투명한 리워드 사용 내역.
  - **직원 확인 모달:** 고객이 실수로 리워드를 사용하는 것을 방지하는 안전장치.
  - **60초 TTL(Time-To-Live):** 요청의 유효 시간을 두어 보안을 강화하는 모습.

### 3.6 후기 섹션 (Review Section)

- **사회적 증거:** "실제 사장님들/사용자들의 후기입니다"라는 제목 아래, 서비스를 사용 중인 실제 사장님과 고객들의 긍정적인 후기를 인용하여 신뢰도를 높입니다.

### 3.7 FAQ 섹션

- **의문 해소:** 자주 묻는 질문들을 모아 답변함으로써, 잠재 고객의 마지막 궁금증까지 해결해 줍니다.

### 3.8 푸터 (Footer)

- **기본 정보:** 회사 정보, 이용 약관, 개인정보처리방침 등 필수 정보를 제공합니다.

## 4. 디자인 및 톤앤매너 (Design & Tone & Manner)

`design-system.md`와 '꾸욱'의 브랜드 아이덴티티를 철저히 준수합니다.

### 4.1 색상 (Color Palette)

- **메인 브랜드 색상:** `kkookk-orange-500` (#ff4d00)을 핵심 요소(메인 CTA, 강조 아이콘 등)에 활용하여 활기차고 긍정적인 인상을 줍니다.
- **사장님 페르소나 색상:** `kkookk-indigo-500` (#2e58ff)을 서브 CTA, 사장님 관련 섹션, 백오피스 이미지 등에 사용하여 전문성과 신뢰감을 더합니다.
- **중립 색상:** `kkookk-navy`, `kkookk-steel` 계열을 텍스트, 배경, 경계선 등에 사용하여 가독성과 안정감을 확보합니다.
- **상태 색상:** `kkookk-green`, `kkookk-red`, `kkookk-amber`는 특정 상태를 나타내는 경우에만 사용합니다.
- `design-system.md`에 정의된 **컬러 토큰(`bg-kkookk-orange-500`, `text-kkookk-navy` 등)**을 일관되게 사용해야 합니다.

### 4.2 타이포그래피 (Typography)

- **메인 폰트:** Pretendard Variable을 사용하여 일관된 가독성과 현대적인 느낌을 유지합니다.
- **폰트 사이즈:** `design-system.md`에 정의된 `kkookk-xs` ~ `kkookk-3xl` 스케일을 활용하여 시각적 계층 구조를 명확히 합니다.
  - 헤드라인: `text-kkookk-3xl` 또는 `text-kkookk-2xl` (`font-bold` 또는 `font-semibold`)
  - 서브 헤드라인: `text-kkookk-xl` 또는 `text-kkookk-lg`
  - 본문: `text-kkookk-base` (`font-normal`)
- **폰트 가중치 (Weight):** Tailwind의 `font-normal`, `font-medium`, `font-semibold`, `font-bold`를 적절히 활용합니다.

### 4.3 간격 및 여백 (Spacing & Layout)

- `design-system.md`의 스페이싱 토큰(`kkookk-sm`, `kkookk-md`, `kkookk-lg` 등)을 사용하여 일관된 레이아웃을 구성합니다. (`space-y-kkookk-md`, `p-kkookk-lg` 등)
- 정보의 밀도를 적절히 조절하여 시각적 피로도를 줄이고, 핵심 요소에 집중할 수 있도록 충분한 여백을 확보합니다.

### 4.4 보더 라디우스 및 그림자 (Border Radius & Shadows)

- `design-system.md`에 정의된 `rounded-kkookk-xl` (버튼), `rounded-kkookk-md` (카드) 등을 사용하여 통일된 디자인 언어를 유지합니다.
- `shadow-kkookk-md` 또는 `shadow-kkookk-lg`를 활용하여 중요한 요소에 시각적 깊이와 계층감을 부여합니다.

### 4.5 시각적 계층 구조 (Visual Hierarchy)

- **가장 중요한 정보 (헤드라인, 메인 CTA, 핵심 가치)는 가장 눈에 띄게** 디자인합니다.
- 색상, 폰트 크기, 여백, 대비를 활용하여 정보의 중요도를 시각적으로 표현합니다.
- 고품질의 이미지, 아이콘, 일러스트를 사용하여 메시지를 강화하고 시각적 흥미를 유발합니다. (예: 사장님 백오피스 스크린샷, 고객 QR 스캔 경험 시뮬레이션)

### 4.6 반응형 디자인 (Responsive Design)

- 모든 기기(데스크톱, 태블릿, 모바일)에서 최적화된 경험을 제공해야 합니다. 모바일 환경에서의 가독성, 터치 영역, CTA 배치 등을 특히 중요하게 고려합니다.

## 5. 기술 구현 가이드 (Technical Implementation Guide)

### 5.1 컴포넌트 재활용 (Component Reusability)

'꾸욱'의 기존 디자인 시스템과 코드베이스를 최대한 활용하여 일관성과 개발 속도를 확보합니다.

- **기본 UI 컴포넌트:** `src/components/ui/` 디렉토리 내의 컴포넌트(예: `Button.tsx`, `Input.tsx`, `Badge.tsx`)를 랜딩 페이지의 기본 빌딩 블록으로 사용합니다.
  - **예시:** 메인 CTA 버튼은 `Button` 컴포넌트의 `variant="primary"` 및 `size="lg"`를 활용하고, 사장님 페르소나 관련 CTA는 `variant="secondary"`를 고려합니다.
- **레이아웃:** 랜딩 페이지 전용의 간결한 레이아웃을 새로 구성하거나, 기존 `CustomerLayout.tsx` 또는 `OwnerLayout.tsx`의 구조를 참고하되, 랜딩 페이지의 목적에 맞게 단순화합니다. (`MobileFrame.tsx`를 통해 제공되는 모바일 쉘 구조를 활용하여 통일감을 줄 수 있습니다.)
- **피처 컴포넌트 (선택):** '꾸욱'의 기능을 설명하는 섹션에서 `StampCardItem.tsx`와 같은 특정 `src/features/**/components` 내의 컴포넌트를 활용하여 실제 서비스 화면의 일부를 보여주는 방식으로 활용할 수 있습니다. 예를 들어, 스탬프 카드 디자인의 유연성을 보여줄 때 `StampCardItem`의 일부를 보여주는 식으로.

### 5.2 데이터 페칭 (Data Fetching)

- 만약 랜딩 페이지에 동적인 데이터(예: 최신 고객 성공 사례 목록)가 필요하다면, `src/app/providers/QueryProvider.tsx`를 통해 제공되는 **TanStack Query (React Query)** 패턴을 따릅니다.
- API 호출은 `src/lib/api/client.ts`와 같은 중앙 집중식 클라이언트를 통해 이루어지는 방식을 권장합니다.
- 랜딩 페이지는 정적인 콘텐츠가 많으므로, 불필요한 데이터 페칭은 지양하고 성능 최적화를 우선합니다.

### 5.3 프로젝트 구조

- 새 랜딩 페이지는 `src/pages/LandingPage.tsx` 또는 `src/features/landing/pages/LandingPage.tsx`와 같이 명확하게 구분된 경로에 위치시킵니다.
- 랜딩 페이지 전용 컴포넌트는 `src/features/landing/components/` 와 같이 해당 피처 내에 캡슐화합니다.

### 5.4 성능 최적화 (Performance Optimization)

- **이미지 최적화:** 모든 이미지는 웹에 최적화된 형식과 크기로 제공하며, 지연 로딩(Lazy Loading)을 적용합니다.
- **코드 스플리팅:** 랜딩 페이지는 초기 로딩 속도가 중요하므로, 필요한 JavaScript만 로드되도록 코드 스플리팅을 고려합니다.
- **CDN 활용:** 정적 자산(이미지, CSS, JS)은 CDN을 통해 제공하여 로딩 속도를 향상시킵니다.

### 5.5 접근성 (Accessibility)

`design-system.md`의 가이드라인을 철저히 준수합니다.

- **색상 대비:** 모든 텍스트와 배경 색상 조합은 WCAG 2.1 AA 기준을 충족해야 합니다. (`WebAIM Contrast Checker` 활용)
- **키보드 탐색:** 모든 대화형 요소(버튼, 링크, 폼 필드)는 키보드로 접근 가능하고 조작할 수 있어야 합니다.
- **포커스 표시:** `focus:ring-2 focus:ring-kkookk-orange-500/30 focus:ring-offset-2`와 같은 표준 포커스 스타일을 사용하여 시각적인 포커스 표시를 명확히 합니다.
- **ARIA 속성 및 시맨틱 HTML:** 적절한 HTML5 시맨틱 태그와 ARIA 속성(예: `aria-label`, `role`)을 사용하여 스크린 리더 사용자에게 올바른 정보를 제공합니다.

## 6. 랜딩 페이지 최적화 모범 사례 (Best Practices)

### 6.1 CTA (Call to Action) 최적화

- **명확하고 간결한 문구:** "지금 바로 무료 체험", "꾸욱 시작하기", "데모 요청" 등 사용자가 얻게 될 가치를 명확히 제시합니다.
- **시각적 강조:** `bg-kkookk-orange-500`과 같은 브랜드 핵심 색상을 사용하여 눈에 띄게 만듭니다. `Button` 컴포넌트의 `size="lg"`를 활용하여 중요도를 높일 수 있습니다.
- **전략적 배치:** Above the Fold 영역에 최소 1개, 그리고 페이지의 핵심 정보 섹션마다 반복적으로 배치하여 전환 기회를 높입니다.
- **대비되는 문구:** 여러 CTA가 있을 경우, 중요도에 따라 색상이나 크기로 차등을 둡니다.

### 6.2 카피라이팅 (Copywriting)

- **문제 해결 중심 헤드라인:** 사장님들의 고충(예: "복잡한 POS 연동", "고객 재방문율 정체")을 언급하고 '꾸욱'이 그 해결책임을 제시합니다.
- **혜택 중심:** 기능 나열보다는 '꾸욱'이 사장님에게 제공할 '재방문율 증가', '간편한 관리', '매출 증대' 등 실질적인 혜택에 초점을 맞춥니다.
- **고객의 언어 사용:** 실제 사장님들이 사용하는 언어와 고민을 반영하여 공감대를 형성합니다.
- **간결하고 명확하게:** 짧고 이해하기 쉬운 문장으로 핵심 메시지를 전달합니다.

### 6.3 시각적 요소 및 구성

- **고품질 비주얼:** '꾸욱' 서비스의 스크린샷, 실제 매장에서 '꾸욱'이 사용되는 모습, 스탬프 카드 디자인 예시 등 고품질의 시각 자료를 활용합니다.
- **브랜드 아이덴티티 일관성:** '꾸욱'의 로고, 색상, 폰트 등 모든 시각 요소가 브랜드 가이드라인과 일치하도록 합니다.
- **소셜 프루프:** 실제 사장님들의 추천사, 성공 사례, 미디어 로고 등을 통해 신뢰를 구축하고 설득력을 높입니다.
- **모바일 최적화:** 모바일 환경에서 랜딩 페이지의 모든 요소가 잘 보이고 작동하는지 철저히 테스트합니다.

## 7. MVP 범위 및 이후 고려사항

### 7.1 MVP (Minimum Viable Product) 랜딩 페이지

- **Above the Fold:** 명확한 헤드라인, 서브 헤드라인, 핵심 비주얼, 메인 CTA.
- **핵심 기능/가치 요약:** 3~4가지 핵심 가치 제안 (예: POS 연동X, 앱 설치X, 마이그레이션, 재방문율). 각 항목은 짧은 설명과 아이콘으로 구성.
- **소셜 프루프:** 1~2개의 사장님 성공 사례 또는 추천사.
- **최종 CTA:** 페이지 하단에 전환을 유도하는 CTA.
- **푸터:** 필수 법적 정보.

### 7.2 추가 고려사항

- **A/B 테스트:** 헤드라인, CTA 문구, 비주얼 등 다양한 요소를 A/B 테스트하여 전환율을 지속적으로 최적화합니다.
- **분석 및 추적:** Google Analytics, Hotjar 등 분석 도구를 연동하여 사용자 행동을 추적하고 개선점을 찾습니다.
- **SEO 최적화:** 검색 엔진 노출을 위한 키워드, 메타 태그, 구조화된 데이터 등을 고려합니다.
