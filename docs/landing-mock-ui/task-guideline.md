# 🎯 추천 작업 방식: 섹션별 반복 개발 (Iterative Development)

## Phase 1: 기반 설정 (한 번만)

먼저 이렇게 요청하세요:

> 랜딩 페이지 기본 구조를 만들어줘.

**포함 내용:**

- 라우팅 설정 (`src/pages/LandingPage.tsx`)
- 레이아웃 뼈대 (`src/features/landing/components/`)
- 공통 컨테이너 스타일
- 섹션 컴포넌트 구조만 (내용은 나중에)

---

## Phase 2: 섹션별 개발 (티키타카 🏓)

각 섹션마다 이런 식으로 요청하면 최고의 결과가 나옵니다.

### 📋 프롬프트 템플릿 (섹션마다 반복)

```
[섹션 이름] 섹션을 만들어줘.

**참고 자료:**
- docs/landing-mock-ui/[해당섹션].png 참고
- docs/landing-page-guideline.md 의 [섹션 번호] 참고

**구현 요구사항:**
1. 디자인: [원하는 특별한 디테일]
2. 인터랙션: [호버, 애니메이션 등]
3. 반응형: 모바일/데스크톱 고려
4. 접근성: 키보드 네비게이션, 포커스 링

**내가 확인하고 싶은 것:**
- [색상 조합이 맞는지?]
- [간격이 적절한지?]
- [카피 문구가 공감되는지?]
```

---

### 실전 예시: Hero 섹션

```
Hero 섹션을 만들어줘.

**참고 자료:**
- docs/landing-mock-ui/header+hero.png 참고
- 종이 스탬프 → 디지털 스탬프 변환을 캐러셀로 보여줘

**구현 요구사항:**
1. 헤드라인: "POS 연동 없이, 바로 시작하는 디지털 스탬프" (더 임팩트 있게 수정 가능)
2. CTA 버튼: "꾸욱 시작하기" (bg-kkookk-orange-500)
3. 캐러셀: 3-4개 슬라이드, 자동 전환 (5초), 인디케이터 표시
4. 애니메이션: 스탬프 찍히는 느낌 (scale effect)

**내가 확인하고 싶은 것:**
- 캐러셀 이미지 placeholder는 어떻게 처리?
- 헤드라인 폰트 크기가 모바일에서 적절한지?
- CTA 버튼 위치가 눈에 띄는지?
```

---

## 🎨 섹션별 핵심 포인트

| 섹션 | 핵심 질문 | 중요 요소 |
|------|-----------|-----------|
| Hero | 3초 안에 가치 전달? | 캐러셀 속도, CTA 색상 |
| Problem | 사장님이 공감할까? | 카피 문구, 아이콘 선택 |
| Solving | UX Flow가 이해되나? | 스텝 간격, 시각적 순서 |
| Security | 신뢰감을 주나? | 뱃지 디자인, 증명 자료 |
| Review | 진짜 후기 같나? | 이름/사진, 별점 표시 |
| FAQ | 가장 궁금한 게 있나? | 아코디언 UX, 내용 완성도 |

---

## 💡 티키타카 팁

### ✅ 좋은 피드백 예시

```
Hero 캐러셀 좋은데, 전환 속도를 3초로 줄이고
pause-on-hover 기능 추가해줘.
그리고 인디케이터를 점(dot) 대신 진행 바(progress bar)로 바꿔봐.
```

### ❌ 피해야 할 요청

- `"전부 다 멋지게 만들어줘"` ← 너무 추상적
- `"이미지는 나중에 넣을게"` ← placeholder 전략 먼저 정해야 함

---

## 🚀 시작할 준비되셨나요?

**제안하는 순서:**

1. **기반 설정** → 라우팅 + 레이아웃
2. **Hero** → 첫인상이 가장 중요
3. **Problem** → 공감대 형성
4. **Solving** → 핵심 가치 전달
5. **Security** → 신뢰 구축
6. **Review** → 사회적 증거
7. **FAQ** → 마지막 궁금증 해소
8. **Footer** → 법적 정보
